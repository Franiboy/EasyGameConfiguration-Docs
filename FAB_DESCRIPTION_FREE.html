This is the free version of EGC - EasyGameConfiguration. All features are included, limited to 10 properties. For unlimited properties (250 per type), get the full version.

An easy-to-integrate Blueprint and C++ library for custom game configurations. Define your settings once in the Project Settings, then read, write, and react to changes from anywhere with a single Blueprint node - no references, categories, or key strings needed.

How It Works

  1. Define your properties in Project Settings (type, default value, display name, description).
  2. Use Get/Set nodes in any Blueprint or C++ class - one node per operation, no setup needed.
  3. Ship - the .ini file is generated automatically. Players can tweak exposed settings, you keep full control over hidden and constant values.

Key Features

  • One-Node Access: Read, write, or subscribe to property changes from anywhere in your project - no object references or string lookups required.
  • Per-Property Control: Save location, display name, description, editability, constant flag, and hidden flag - all configurable per property directly in the editor.
  • Save & Load: Create, load, reset, and unload configuration files via Blueprint, with native multi-save support (e.g. per-player or per-server configs).
  • Auto-Generated .ini Files: Configuration files are created and managed automatically - no manual file handling needed.
  • Auto-Generated Settings UI: Render UI widgets for all your properties with a single function call. Each property can override its widget style or type individually.
  • Tamper Protection: Optional hash-based validation detects when players manually edit save data. You define per-property fallback values that are used when tampering is detected.
  • 9 Data Types: Boolean, Integer, Float, String, Vector2D, Vector, Color, Rotator, Transform.

Free vs Full Version

  • Free: All features, limited to 10 properties. Includes 10 pre-configured example properties to get started immediately.
  • Full: Up to 250 properties per type. Available here.

Use Cases

  • Game Settings - Let players adjust difficulty, graphics, audio, or gameplay parameters through an auto-generated .ini file.
  • Server Configurations - Modify spawn rates, player limits, or world rules per server without rebuilding.
  • Hidden Properties - Store values that only your code can read and write - invisible in the .ini file. Useful for internal state, progression data, or save-game-like features.
  • Constants - Define read-only values accessible from anywhere in the project. Players can see them in the .ini but cannot change them.

Code Modules

  • EasyGameConfiguration [Runtime] - All get/set/reset/delegate/widget functionality. Editor-only features (settings UI, Blueprint node refresh) are automatically excluded from packaged builds.
